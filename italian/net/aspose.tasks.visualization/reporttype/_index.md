## ReportType enumeration

Tipo di report grafico del progetto.

```csharp
public enum ReportType
```

### I valori

| Nome | Valore | Descrizione |
| --- | --- | --- |
| ProjectOverview | `0` | Mostra la data di inizio e fine del progetto, percentuale di durata completata, percentuale di completamento per attività di primo livello e traguardi in arrivo. |
| CostOverview | `1` | Mostra le date di inizio e fine del progetto, il costo corrente pianificato e rimanente, % di completamento e valori di costo per le attività di primo livello. |
| WorkOverview | `2` | Mostra la linea di base, il lavoro effettivo e rimanente per ogni attività di primo livello e il lavoro per le risorse di lavoro. |
| ResourceOverview | `3` | Mostra il lavoro di base, effettivo e rimanente per risorsa. |
| ResourceCostOverview | `4` | Mostra il costo baseline, effettivo e rimanente per risorsa. |
| CriticalTasks | `5` | Mostra le attività di progetto critiche. |
| LateTasks | `6` | Mostra le attività del progetto in ritardo. |
| Milestones | `7` | Mostra le pietre miliari in ritardo, in arrivo e completate. |
| UpcomingTask | `8` | Mostra le attività in scadenza durante la settimana corrente e le attività che iniziano durante la settimana corrente. |
| CostOverruns | `9` | Mostra la varianza dei costi per attività e risorsa. |
| TaskCostOverview | `10` | Mostra il costo di base, effettivo e rimanente di tutte le attività di primo livello. |
| OverallocatedResources | `11` | Mostra il numero di ore di lavoro rimanenti per risorse allocate in eccesso. |
| SlippingTasks | `12` | Mostra le attività che devono terminare dopo la data di fine prevista (è necessario impostare la previsione). |
| BestPracticeAnalyzer | `13` | Mostra le attività senza lavoro effettivo, le attività non assegnate, le attività con durata inferiore a 8 ore e i riepiloghi assegnati con le risorse. |
| Burndown | `14` | Include grafici di burndown del lavoro e task burndown. Il grafico di burndown del lavoro mostra quanto lavoro hanno finito le persone, quanto è pianificato per essere terminato prima della data di fine del progetto, e la stima di base di quanto lavoro sarebbe stato completato a questo punto nel progetto. Il diagramma di burndown delle attività mostra il numero di attività completate, il numero rimanente, e la stima di base di quante sarebbero state completate a questo punto del progetto. |
| CashFlow | `15` | Mostra i costi e i costi cumulativi per trimestre per tutte le attività di primo livello. |
