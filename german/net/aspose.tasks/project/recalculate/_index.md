Plant alle Projektaufgaben-IDs, Gliederungsebenen, Start-/Endtermine neu, legt frühe/späte Termine fest, berechnet Lücken, Arbeits- und Kostenfelder mit optionaler Validierung.


| Parameter | Typ | Beschreibung |
| --- | --- | --- |
| validate | Boolean | Wenn wahr, wird die Validierung der Neuberechnung durchgeführt. Welche Daten werden validiert: Im Moment ist nur eine grundlegende Validierung der Datumsbereiche von Aufgaben und Aufgabenverknüpfungen implementiert. Die Datumsbereiche von Aufgaben (z. B. ActualStart - ActualFinish, EarlyStart - EarlyFinish usw. ) sowie Aufgabenverknüpfungsdaten werden anhand der Datumskriterien überprüft, dass das Startdatum kleiner oder gleich dem Enddatum ist. Wenn eine der oben beschriebenen Bedingungen fehlschlägt, dann[`RecalculationValidationException`](../../recalculationvalidationexception) wird geworfen. |
